由于本地环境缺失 Node.js/npm 环境，无法直接构建 EXE/APK。我们将采用 **"Cloud-First"（云端优先）策略**，通过配置 GitHub Actions 自动化流水线来完成所有跨平台构建任务。

### 核心架构：PWA + GitHub Actions

1.  **Web端 (PWA)**：
    *   **方案**：直接作为 Progressive Web App 运行，支持全平台（iOS/Android/Desktop）安装。
    *   **实现**：新增 `manifest.json` 和 `service-worker.js`，配置图标。
    *   **部署**：使用 GitHub Actions 自动部署到 GitHub Pages，提供 HTTPS 和 CDN 加速。

2.  **桌面端 (Windows/macOS)**：
    *   **方案**：Electron 封装。
    *   **实现**：创建 `package.json` 和 `main.js`（Electron 入口），但**不**在本地安装依赖。
    *   **构建**：编写 GitHub Workflow (`build-desktop.yml`)，在云端安装依赖并运行 `electron-builder`，生成 `.exe` (Windows) 和 `.dmg` (macOS) 并发布到 GitHub Releases。

3.  **移动端 (Android)**：
    *   **方案**：PWA (TWA) 或 Webview。鉴于环境限制，首选 PWA 引导安装。
    *   **构建**：若需 APK，可在 GitHub Actions 中集成 `Bubblewrap` 或简单的 Android 构建流（可选，本次主要通过 PWA 覆盖）。

4.  **监控与更新**：
    *   **Sentry**：通过 CDN 方式直接引入 SDK，无需本地构建。
    *   **自动更新**：PWA 自带更新机制；Electron 应用配置 `electron-updater`（指向 GitHub Releases）。

### 执行计划

1.  **PWA 改造**：创建 `manifest.json`、`sw.js`，并在 `index.html` 中注册。
2.  **Sentry 集成**：在 `index.html` 头部加入 Sentry CDN 代码。
3.  **Electron 配置**：创建 `package.json`（定义依赖和构建脚本）和 `electron-main.js`。
4.  **CI/CD 流水线**：
    *   创建 `.github/workflows/deploy_web.yml`：自动部署到 GitHub Pages。
    *   创建 `.github/workflows/build_apps.yml`：自动构建 Electron 应用并发布 Release。
5.  **文档交付**：
    *   创建 `DEPLOY.md`，包含 Web 访问链接（占位符）、各平台安装指南、二维码生成指引。

*注：您只需将这些配置文件提交到 GitHub 仓库，GitHub Actions 将自动接管后续的构建与发布工作。*
