此计划旨在通过全方位的技术优化，将“2026 步步高升”打造为一款商业级质量的轻量化游戏。

## 1. 核心架构升级：性能与渲染
针对低端设备（如千元机、旧款 iPhone）进行深度优化，确保稳定 30-60 FPS。
-   **离屏渲染缓存 (Offscreen Caching)**: 重构 `Assets` 模块，将所有矢量图形（福马、云朵、灯笼）在游戏启动时预渲染到内存中的离屏 Canvas。主循环中仅执行 `drawImage` 操作，替代昂贵的 `ctx.ellipse` 和 `shadowBlur` 指令。此举预计降低 60% 的 GPU 开销。
-   **DPR 自适应系统**: 自动检测设备像素比 (`devicePixelRatio`)。
    -   **高端机**: 开启 2x/3x 渲染，保证视网膜屏清晰度。
    -   **低端机**: 智能降级至 1x 渲染，并关闭粒子特效，优先保帧率。
-   **垃圾回收优化 (GC Optimization)**: 优化粒子系统，使用对象复用池代替频繁的 `new` 和 `splice`，减少游戏过程中的卡顿（Jank）。

## 2. 游戏性调优：平滑难度曲线
重构关卡生成算法，从“纯随机”改为“动态难度控制 (DDA)”。
-   **阶段一（新手期 0-1000分）**: 平台宽大且密集，无陷阱，确保新手能轻松获得正反馈。
-   **阶段二（进阶期 1000-3000分）**: 引入移动云朵，间距适度拉大。
-   **阶段三（挑战期 3000+分）**: 引入破碎云朵和稀疏布局，考验精准操作。
-   **物理手感微调**: 增加“边缘容错”和空中惯性控制，优化左右移动的加速度曲线，使操作更跟手。

## 3. 交互与引导：零门槛上手
-   **交互式引导**: 首次进入游戏时，前 5 个平台将生成为固定的“安全阶梯”，并显示动态的手指滑动动画，引导玩家操作。
-   **UI 适配**: 针对刘海屏和不同长宽比设备（iPad vs 超长屏手机）优化 UI 布局，确保按钮和分数始终在安全区域内。

## 4. 实施路线图
1.  **渲染重构**: 修改 `js/assets.js`，实现 Cache 系统。
2.  **引擎升级**: 修改 `js/game.js`，加入 DPR 支持和 FPS 监控降级逻辑。
3.  **玩法调整**: 重写 `generatePlatform` 算法，加入难度分级。
4.  **UI/UX**: 优化 CSS 响应式规则，添加引导动画。
5.  **测试与交付**: 编写性能测试报告和兼容性文档。
